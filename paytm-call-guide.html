Questions to Ask Paytm
Question 1 — Webhook / Callback Support
Does Paytm send automatic webhook/callback notifications after every POS transaction?
"Does Paytm send automatic webhook/callback notifications to our URL after every POS transaction? Or do we need to rely only on the Status Enquiry API polling? We have already configured a callback URL: https://samedaysolution.in/api/paytm/notification/lagoon — can you enable this to receive real-time transaction notifications?"
Question 2 — Channel ID Confirmation
What exact channelId value should we use?
"What exact channelId value should we use in our API requests? Your email says EDC, but the documentation example shows RIL. Please confirm."
Question 3 — Checksum Generation
Confirm the checksum method for POS/ECR integration.
"For checksum generation — we use the AES/Merchant Key with the paytmchecksum Node.js library on the body JSON string. Is that correct for POS/ECR integration? Or is there a different method?"
Question 4 — Callback Payload Format
Request sample webhook payload for success and failure.
"When a transaction completes on the POS, what is the exact JSON payload format sent in the webhook callback? Can you share a sample success and failure payload via email?"
Question 5 — Transaction Types Coverage
Do notifications cover Void, Refund, and Settlement?
"Besides Sale transactions, does the notification also cover Void, Refund, and Settlement events?"
Question 6 — Multiple Devices / MIDs
How does multi-merchant, multi-device routing work?
"In production, we will have multiple merchants with multiple POS devices each. Will each store get a separate MID and TID? Can we use the same callback URL for all, with merchant-specific path routing?"
Question 7 — Settlement API
Is there an API to fetch daily settlement data?
"Do you provide a settlement API or settlement report API to fetch daily settlement data programmatically? Or is settlement only visible on the Paytm dashboard?"
Question 8 — Status Enquiry Retry Logic
Max retries and timeout for Status Enquiry polling.
"For the Status Enquiry API — your docs say poll after 10 seconds with 10-second intervals. What is the maximum number of retries? After how long should we consider the transaction timed out?"
Question 9 — Void / Abort API
Is there a server-side Void/Cancel API?
"Is there a Void API for cancelling a transaction from our server? The email mentioned an Abort API doc was attached — can you walk us through it?"
Question 10 — Go-Live Checklist
What test cases are required before production?
"What exactly is in the production checklist? What test cases do we need to pass before going live?"
Bonus Points to Raise
Postman Collection
Ask if they have a Postman collection for the POS APIs — makes testing much faster.
URL Clarification
Confirm if securegw-stage.paytm.in and securestage.paytmpayments.com are the same endpoint (docs reference both).
IP Whitelisting
Ask if your server IP needs to be whitelisted for API access.
Rate Limits
Ask about rate limits on the Sale API and Status Enquiry API.
If Paytm Asks You — Answers Ready
If they ask
"What is your tech stack?"
"We run a Next.js application on Node.js, hosted on Vercel, with Supabase (PostgreSQL) as our database. Our backend uses serverless API routes."
If they ask
"What is your callback/notification URL?"
https://samedaysolution.in/api/paytm/notification/lagoon
It accepts POST requests with JSON body. We return HTTP 200 with a JSON acknowledgment.
If they ask
"What is your use case / business model?"
"We are a POS aggregation platform. We deploy POS machines to retailers through our distributor network. We need real-time transaction data to credit retailer wallets, track settlements, and provide a transaction dashboard to our merchants."
If they ask
"Do you need to initiate payments from your server?"
"Yes, we want both:
• Server-initiated payments via Sale API (our billing system sends amount to POS)
• Standalone POS transactions with webhook callbacks (customer pays directly on POS, we get notified)"
If they ask
"How many devices/merchants will you have?"
"Currently one device for Lagoon Craft Labs on staging. In production, we plan to scale to multiple merchants with multiple devices per merchant. Each merchant will have its own MID."
If they ask
"Are you PCI DSS compliant?"
"We do not store card numbers or sensitive cardholder data. We only store masked card numbers, RRN, and transaction metadata received from Paytm callbacks. Card processing happens entirely on the Paytm POS terminal."
If they ask
"What integration mode?"
"Wireless (cloud-based ECR). Our server communicates with the POS via Paytm's cloud APIs over the internet."
If they ask
"Have you activated the staging device?"

If not done: "We are in the process. We have the staging credentials and will complete activation today."
If they ask
"What payment modes do you need?"
"All modes — Card (Visa, MasterCard, RuPay), UPI/QR, and any other modes supported by the terminal."
If they ask
"Do you need EMI support?"
"Not immediately, but we'd like to keep the option open for the future."
